Cross-Site Scripting

Cross-site Scripting is malicious script that allows hackers to insert into a server. It allows hackers to get many victims as they are able to execute attacks on big companies and do it in ways that manipulate many people. Once a hacker has gotten through, they are able to act as the user and gain access to the usersâ€™ sensitive data. There are many different types of Cross-Site Scripting, reflective, stored and DOM-based. Each one of the methods are used depending on what the hacker is trying to use to obtain the users data. Reflected XXS is when a malicious script is placed within a website and convinces the user into executing it like by clicking on a link. We are seeing this commonly now with phishing attempts. Phishing attempts are sometimes extremely hard to recognize, we see them often mimic emails you would usually get. For instance, if you typically get emails from Netflix when you need to update your payment details hackers will change slight things like using a .com email instead of a .org email in order to trick people into thinking they are actually updating their information with Amazon. Stored XXS is where the attacker injects code into a web application. When the user loads the page that is when the attack is triggered. DOM-Based XXS is where a hacker gains access to the users side by manipulating the Document Object Model of a page. This type of scripting is the hardest to do but also the hardest to fix.
